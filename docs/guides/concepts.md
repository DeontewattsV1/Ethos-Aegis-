# Concepts

This template demonstrates a **living documentation** workflow where source code is the single source of truth and the README is regenerated by CI.

## The contract

There are three artifacts and one rule:

| Artifact | Source of truth |
|---|---|
| `examples/*.ts` | The actual code, authored by humans. |
| `README.md` code blocks | Mirror of `examples/*.ts`, written by `scripts/sync-readme.ts`. |
| `docs/output-snapshots/*.txt` | Captured stdout of each example, written by `scripts/run-examples.ts`. |

**Rule**: a stale README is a CI failure, not a documentation debt item.

## Region markers

The README never contains source-of-truth code. Each code block is preceded by a marker:

```md
<!-- example:basic/01-subscribe.ts -->
```ts
// contents of examples/basic/01-subscribe.ts (replaced by CI)
```
```

And each rendered-output block has its own marker:

```md
<!-- output:basic/01-subscribe.ts -->
```
captured stdout
```
```

## Data flow

```
examples/*.ts  ──►  sync-readme.ts   ──►  README.md code blocks
examples/*.ts  ──►  run-examples.ts  ──►  docs/output-snapshots/
docs/output-snapshots/ + README.md   ──►  readme.yml CI validation
```

## Adding a new example

1. Drop a `.ts` file under `examples/basic/`, `examples/advanced/`, or `examples/interactive/`.
2. Add a region marker block in `README.md`:
   ```md
   <!-- example:basic/my-new-example.ts -->
   ```ts
   ```
   <!-- output:basic/my-new-example.ts -->
   ```
   ```
   ```
3. Run `make docs`. The contents are filled in automatically.
4. Commit and open a PR.

## Why this matters

The traditional alternative is "documentation as a separate document" — which goes stale silently. Here, drift is an error CI surfaces immediately.
